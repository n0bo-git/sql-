# 体育用品批发销售信息系统

## 项目概述

本系统旨在为一家经营体育用品批发的商店提供全面的销售信息管理，涵盖顾客信息、生产厂家信息、库存信息等。系统分为前台（顾客使用）和后台（管理员使用）两大模块，支持在线订购、订单处理、库存管理、报表生成等功能。

## 功能要求

### 前台模块（顾客使用）

- 订购商品
- 请求发票
- 填写邮寄信息（收件人、地址、联系电话、订单号等）
- 提供支付信息

### 后台模块（管理员使用）

- **订单处理**：确认用户订单、发货、打印发票（如缺货则进行缺货处理）、打印邮寄信息
- **库存管理**：维护库存信息，库存不足时及时联系生产厂家进货
- **报表生成**：
  - 已成功完成订单列表
  - 未付款订单报表
  - 未发货订单报表

## 主要数据项

### 1. 顾客信息
- 顾客代码
- 顾客名称
- 姓名
- 地址
- 邮编
- 电话

### 2. 订单信息
- 订单号
- 下单日期
- 顾客代码
- 运输要求
- 能否供货标志
- 运输日期
- 货物重量
- 运费
- 付款清单

### 3. 订单细则
- 细则编号
- 订单号
- 产品分类编号
- 生产厂
- 数量
- 总金额

### 4. 库存商品信息
- 商品分类编码
- 生产厂家编码
- 商品说明
- 单价

### 5. 生产厂家信息
- 厂家代码
- 厂家名称

## 技术实现建议（可选）

- 前端：可根据需要选用 Vue/React 等框架
- 后端：Spring Boot / Django / Node.js 等
- 数据库：MySQL / PostgreSQL
- 接口风格：RESTful API
