六、修复 Mapper-Controller-前端交互问题
================================================================================

6.1 request.js 重写：
  问题：
    - fetch 网络错误无诊断信息
    - HTTP 4xx/5xx 直接 response.json() 可能崩溃
    - params 对象混入 fetch() options
  修复：
    - 分离 params → URL 查询参数，不传入 fetch()
    - 网络错误 → "无法连接后端服务，请确认 Spring Boot 已启动"
    - HTTP 非 2xx → "HTTP {status}: {body}"
    - JSON 解析失败 → "后端返回格式异常，无法解析 JSON"
    - 业务 code ≠ 200 → 抛出 data.msg

6.2 GoodsMapper 重写：
  问题：
    - GoodsVO 非 @TableName 实体，列名隐式映射可能失败
    - <script><where> 动态 SQL 行为不确定
  修复：
    - 去掉 <script><where>，改用纯 SQL：
      WHERE (#{keyword} IS NULL OR ... OR goods_desc LIKE ...)
      AND   (#{categoryId} IS NULL OR ... OR category_id = ...)
    - 添加 @Results 显式映射 7 列 → GoodsVO 属性 (goodsCode/manuCode/
      categoryId/goodsDesc/unitPrice/manuName/coverImage)

6.3 前端错误诊断增强：
  - loadGoods() catch 块加入 console.error('[加载商品]', e)
